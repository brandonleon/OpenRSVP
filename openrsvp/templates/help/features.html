How Ranking & Decay Work

Events and channels start with a baseline score. Each time an event link is opened, the event's last_accessed timestamp updates; channel pages (and events within a channel) refresh last_used_at. The decay cycle applies exponential decay based on the time since the last activity, so frequently visited links stay near the top while idle items drift down.

  • • Public listings hide events once scores drop below the hide threshold and they age past the hide window, but direct links still work.
  • • Events are eligible for deletion after the post-event grace window, after the delete-after age, and only when their score falls below the delete threshold (upcoming events can be protected).
  • • Channels decay by last_used_at and are only removed when empty and below the delete threshold.
  • • RSVPs and messages inherit the event lifecycle and are removed when the event is purged.
  • • Decay and SQLite vacuuming run on a configurable schedule.
